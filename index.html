Guideline to run the COMOT demo on a single host

This guide is going to introduce the step of running COMOT components on a single machine such as one VM or on user's PC.

  • Install and run COMOT dashboard
  • Create a HelloElasticity application by COMOT fluent API
  • Deploy the application by SALSA
  • Control the application by SYBL
  • Monitor the application by MELA
  • Analyze the elasticity by QUELE

Materials

User can download all the built version of COMOT modules on our temporary repository: http://128.130.172.215/repository/. The repository is also shown in the dashboard after the installation.

Prerequisites

  1. Operation system: This demo was tested with Ubuntu 14.04. As the application need Docker to run, our Docker installation does not work with older Ubuntu. User can install Docker manually if they use older system (Please refer: https://docs.docker.com/installation/ubuntulinux/)
  2. Root privilege: This demo need root to run Docker.
  3. RAM: At least 2GB recommended for running all the COMOT modules, ideally more than 4GB.
  4. Storage: At least 1GB free of disk to store the Docker local image.
